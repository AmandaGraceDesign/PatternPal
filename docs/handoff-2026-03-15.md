## What Was Done This Session

### 1. Fixed Cross-Tab Data Transfer (was BLOCKED)
- Replaced broken sessionStorage approach with postMessage retry handshake
- Child tab sends `ready` every 200ms, parent responds with `init` containing image data URL
- postMessage uses structured cloning — no storage quota limits
- Handles React strict mode double-mounting and timing issues
- Files: `src/lib/seam-inspector/openSeamInspector.ts`, `app/seam-inspector/page.tsx`

### 2. Relative Zoom System
- Main canvas zoom is now relative: fit-to-viewport = 100%
- `baseZoom` state stores the fit-to-viewport value calculated on upload
- Slider range: 50%–200% (display values), actual zoom = `(displayZoom / 100) * baseZoom`
- Prevents users from zooming in enough to inspect seams on the main canvas (pro feature gate)
- Files: `app/page.tsx`, `src/components/layout/PatternControlsTopBar.tsx`

### 3. Scale Preview Fix
- Canvas rendering now uses effective tile dimensions (`tileWidth / naturalWidth`) instead of static DPI
- When scale preview changes tile size, the canvas correctly re-renders at the new scale
- Rulers also update to reflect effective DPI
- File: `src/components/canvas/PatternPreviewCanvas.tsx`

### 4. Hairline Gap Fix (Both Canvases)
- `Math.ceil` on tile dimensions, `Math.round` on all tile positions
- Applied to `PatternTiler.ts` (main canvas) and `SeamInspectorCanvas.tsx`
- Eliminates sub-pixel anti-aliasing gaps between tiles

### 5. Seam Inspector UX
- Opens with tile top-left corner centered in viewport (was tile center)
- Floating zoom control is draggable — grip dots handle on left side
- Event propagation stopped so dragging the control doesn't pan the canvas
- "File Name:" label added before filename in main controls
- File: `src/components/analysis/SeamInspectorCanvas.tsx`
